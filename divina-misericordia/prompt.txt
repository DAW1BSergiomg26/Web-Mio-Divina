Tu tarea es realizar una auditoría completa del frontend del proyecto y aplicar mejoras seguras para optimizar la calidad del código, el rendimiento y la mantenibilidad del sitio web.

Debes trabajar con extrema precaución para no romper el layout, las funcionalidades existentes ni la estructura del proyecto.

OBJETIVO PRINCIPAL

Analizar y optimizar el frontend del proyecto para:

mejorar rendimiento
reducir CSS innecesario
evitar duplicación de estilos
mejorar accesibilidad
mejorar responsive design
optimizar animaciones
mejorar estructura HTML
reducir complejidad del CSS
hacer el código más mantenible

FASE 1 — ESCANEO COMPLETO DEL PROYECTO

Antes de modificar nada debes analizar completamente el proyecto.

Debes revisar:

estructura de carpetas
archivos HTML
archivos CSS
archivos JS si existen

Detectar:

clases CSS duplicadas
reglas CSS redundantes
estilos no utilizados
animaciones innecesarias
selectores demasiado complejos
posibles conflictos CSS
errores HTML
problemas de accesibilidad
problemas de responsive design

No realizar cambios hasta terminar este análisis.

FASE 2 — AUDITORÍA FRONTEND

Después del análisis debes identificar:

Problemas en la estructura HTML

Problemas en la arquitectura CSS

CSS duplicado o redundante

CSS que podría simplificarse

selectores demasiado complejos

animaciones innecesarias o pesadas

problemas de responsive

problemas de accesibilidad

posibles mejoras SEO en el HTML

posibles mejoras de rendimiento

FASE 3 — REFACTORING SEGURO

Después de la auditoría debes aplicar refactorización segura.

Reglas obligatorias:

no romper el layout existente
no eliminar clases usadas
no modificar funcionalidades existentes
no modificar textos de la web
no eliminar archivos importantes

Debes:

simplificar CSS
eliminar duplicación de estilos
mejorar nombres de clases si es necesario
organizar mejor el CSS
optimizar animaciones
reducir peso del CSS
mejorar estructura semántica HTML
mejorar accesibilidad

FASE 4 — OPTIMIZACIÓN DE RENDIMIENTO

Optimizar:

tamaño del CSS
selectores CSS
reutilización de clases
animaciones CSS
uso de transform en lugar de propiedades costosas
uso correcto de clamp() para responsive
estructura del DOM

Si detectas animaciones pesadas debes optimizarlas.

FASE 5 — MEJORAS DE ACCESIBILIDAD

Revisar:

uso correcto de etiquetas semánticas
atributos aria
contraste de colores
estructura de encabezados h1 h2 h3
legibilidad del texto
focus states
interacción hover y teclado

Aplicar mejoras si es necesario.

FASE 6 — RESPONSIVE DESIGN

Revisar el comportamiento en:

móvil
tablet
pantallas grandes

Verificar:

uso de clamp()
uso correcto de max-width
layout flexible
tipografías fluidas

Optimizar si es necesario.

FASE 7 — VALIDACIÓN FINAL

Después de aplicar cambios debes validar:

HTML válido
CSS válido
no hay errores de sintaxis
layout intacto
animaciones funcionando
responsive correcto

Si detectas errores debes reportarlos.
